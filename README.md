# VOC 콘솔 — VOC 전달 자동화 & 대시보드

CS가 레드마인 VOC를 이메일로 수동 전달하던 비효율을 없애고, 전용 입력 화면과 UX·PM 공용 대시보드로 한 번에 처리하는 내부 콘솔입니다. (Redmine PRD v1.1 기반)

## 화면 구성
- **CS 입력** — VOC 본문 입력, 모델명(선택), 고객 출처(국내/해외), 레드마인 티켓 번호(선택, 원본 링크 자동 생성), 임시저장, 제출 시 접수번호 자동 부여
- **대시보드 (UX·PM 공용)** — VOC 목록, AI 요약·분류(면책 표시), UX 수동 보정(→검토 완료 전환), 유형/영향범위/모델/출처/감정 필터, 반복 이슈 감지 배지, 우선순위 태깅(High/Mid/Low), 상세 보기, PM 개발 전달 메모 + 상태(검토중/개발 요청/완료). PM도 UX와 동일하게 전체 내용을 봅니다.

## 로컬 실행
정적 사이트입니다. 별도 빌드 없이 그대로 엽니다.

```bash
# 아무 정적 서버나 사용
python3 -m http.server 8080
# → http://localhost:8080
```

## GitHub Pages 배포
1. 이 파일들(`index.html`, `styles.css`, `app.js`)을 저장소 루트에 올립니다.
2. **Settings → Pages → Build and deployment → Source: Deploy from a branch**
3. Branch를 `main` / `(root)`으로 지정하고 저장하면 게시됩니다.

## 알아둘 점
- **데이터 저장**: 백엔드가 없는 v1이라 데이터는 브라우저 `localStorage`에 저장됩니다(해당 기기/브라우저 한정). 다중 사용자 공유·1년 보존 요건은 서버 도입 시 충족됩니다.
- **"AI 요약·분류"는 키워드 기반 휴리스틱 대체 구현**입니다. 실제 LLM이 아니며, 운영 시 서버 측 AI 연동으로 교체해야 합니다. (대체 구현이어도 PRD의 AI 면책 표시 요건은 그대로 적용했습니다.)
- **레드마인 원본 URL**: `app.js`의 `REDMINE_BASE` 상수를 실제 레드마인 주소로 바꾸세요.
- **필터 공유**: 필터 상태가 URL 해시에 반영되어 링크로 공유할 수 있습니다.
